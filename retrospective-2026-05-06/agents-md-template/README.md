# `agents-md-template` — session 3 AGENTS.md additions

## Why this directory

This directory holds the session-3 AGENTS.md harvest as a separate
spec-of-its-own. The companion `SPEC.md` lists 12 do/don't rules
ready to drop into the project's `AGENTS.md` (or another agent-facing
playbook). Each rule is grounded in a specific session-3 event with a
one-line provenance phrase, so the user can evaluate them
individually.

These rules are **proposed**, not auto-applied. The user (or the next
agent acting on the user's instruction) should review each rule and
decide whether to accept, amend, or reject before adding it to
`AGENTS.md`.

## How to consume

1. Read `SPEC.md` end-to-end.
2. For each rule, decide:
   - Accept verbatim → drop into `AGENTS.md` under the right section.
   - Amend → fold into an existing rule with a slight wording change.
   - Reject → record the rationale in a comment so the next session doesn't re-propose it.
3. When applying, prefer minimal additions. The existing `AGENTS.md`
   has its own rhythm; rules should integrate, not pile up.

## Cousin

`retrospective/agents-md-template` (session 2) holds the prior
session's harvest. Reference it for tone and shape; this directory
adds material from the live-testing experience that session 2's
harvest could not have included.
