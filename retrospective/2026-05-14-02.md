## Part 4 — proposed ADRs

Architectural decisions made (or reaffirmed) during this session that the user may want to record as formal ADRs. **Titles only.** Use the `adr` skill to author any.

- **Subagent parallelism cap: dispatch all in one message when MAX_PARALLEL ≥ N rather than waving** — Reaffirmed today by dispatching 6/6 in a single message with zero conflict, vs PLAN-PACKAGE.md's 4+2 wave suggestion.
- **Commit the recipe install.md but gitignore the tarball** — Picked during Phase 6; recipe is diffable Markdown (~756 KiB), tarball is opaque binary (~200 KiB). Codifies the trade-off for future bundle builds.
- **Rooted `/dist/` over bare `dist/` in repo .gitignore** — Made today. The bare form excludes a *directory at any depth*, which prevents file-level exceptions inside. Worth documenting as a convention.
- **Subagents writing to a new top-level `lib/` must use `git add -f` (not a nested `.gitignore`)** — Three subagents invented a nested-gitignore workaround in this session; two used `-f`. Inconsistency cost a cleanup commit. Pick one and document it.
- **Bundled-templates contract: byte-for-byte copies, contract-tested** — Reaffirmed by today's `test_template_parity.py`. The convention is in SPEC-PACKAGE.md; an ADR would surface it for projects beyond this one.
- **Text-content recipe encoders must preserve trailing newlines** — Surfaced today; cost 190 mismatches in `test_archive_round_trip` before the fix. Worth a small ADR if recipe-style packaging recurs.
